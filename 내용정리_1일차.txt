streamlit 데이터의 특징 
-> 내가 어떤 선택한 데이터 
-> st.radio/ st.check 등등 
-> 각각 데이터는 새로 함수를 실행하면 이전 데이터가 사라짐 
-> 그래서, 내가 입력한 데이터가 이 페이지 나가기 전까지 사라지지 않게 하려면 
-> 그걸 항상 딕셔너리(저장소)로 저장해야함 
-> 여기에 저장된 정보는 - 페이지가 이동하기 전까지는 저장소로, 계속 저장된다 

st.session_state라는 객체가 있고 -> 이거는 딕셔너리 형태로 저장해줄 수 있는 저장소다 라고 생각 

-> 변수값을 session_state라는 저장소에 저장하지 않으면 
st.~ 이벤트가 계속 발생할 때 변수가 계속해서 초기화가 된다 


# 상태 유지 예시
if "count2" not in st.session_state:
    st.session_state.count2 = 0

-> 만약 st.session_state 안에 count2라는 키 값이 등록이 안되어져 있으면 
-> 일단 먼저 등록을 해줘라 

-> 내가 계속해서 쓰고싶은 데이터가 있으면 
-> st.session_state.변수명 또는  st.session_state[변수명] 이렇게 선언을 해서, 저장소에 넣어줘야 한다
