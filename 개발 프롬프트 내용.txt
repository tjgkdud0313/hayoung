나는 기획팀 사람이고, 전산 상에 변경사항이 필요할 때 
개발팀에 개발 전산개발요청서 (SR) 피피티 파일을 만들어서 전달하는 
기획팀에서 일하고 있어 


나는 지금 이 전산개발요청서를 만드는 챗봇 사이트를 하나 만들고 싶어
(챗봇이 아니라,그냥 폼을 채워넣고, 그 폼에서 제출만 하면 되는 화면이여도 돼)

사용자가 해당 전산개발요청서에 필요한 이미지를 첨부하고 
필요한 내용들을 넣으면, 그 입력값들이 슈퍼베이스 데이터베이스 상에 생성되고  
백엔드단에선 이제 그 데이터베이스 (INPUT 테이블) 그 한 줄을 이제 읽어서
그 내용들을 피피티를 만드는 툴로 보내서 API를 쏘든 피피티를 생성하게 해줄거야


사용자에게 입력받을 인풋은 아래와 같아
(1)분류
Q1.어떤 전산을 개발하시나요? 
- UNIS(웹) / 채널(앱) 중에 사용자가 하나 클릭할 수 있었으면 좋겠어 


(2)화면명  
Q2.어떤 화면을 개발하시나요? 
- 사용자가 입력하는 화면명칭을 넣어줘 
EX)대출실행(PL)

(3)과제명 
Q3.과제명을 간단하게 입력해주세요. 
- 사용자가 입력하는 과제명을 넣어줘 
EX) 압류금지 생계비계좌 개설해지정보 온라인조회 요청 


(4)과제설명 
Q4. 과제에 대한 설명을 간단하게 작성해주세요. 
- 사용자가 입력하는 과제설명을 넣어줘 
EX) 대출실행 화면에서 계좌인증 버튼 클릭시, 해당 계좌가 압류금지 생계비계좌인지 확인하는 로직 추가 


(5)요청배경
Q5. 개발을 요청하시는 이유가 무엇인가요? (목적) 
- 사용자가 입력하는 개발의 요청 이유, 배경,목적들을 텍스트로 넣어줘 
EX) 대출 송금을 위해 입력한 사용자의 계좌가 '압류금지 생계비계좌'일 경우 250만원 이상 입금이 불가하다(잔액 250만원 초과x) 
따라서 송금액의 일부가 불명예금으로 반환되는 오류가 생기는데, 이를 사전에 방지하고자 계좌를 입력하고 인증 버튼을 클릭했을 시 
기존 계좌존재여부 인증 로직과 더불어 동시에 신용정보원 API를 호출하여 사용자가 압류금지 생활비계좌를 입력하는 것을 방지한다 

(6)변경사항 
Q6. 구체적인 개발 요건이 무엇인가요? 
EX) 계좌 인증 버튼을 클릭했을 때, 기존 계좌존배여부 확인 로직 + (2) 신규 '압류금지 생활비계좌' 확인 로직 
이렇게 2가지의 인증을 동시에 진행할 수 있도록 한다.
만약 해당 계좌가 생계비계좌라면 팝업 안내메시지 '해당 계좌는 압류금지 생계비계좌입니다. 다른 계좌를 입력해주세요'가 뜨면서
팝업을 닫았을 때, 기존에 INPUT창에 입력했던 계좌번호와 계좌명 예금주가 다시 빈칸(초기화)된다 


(7)요청자 (8) 요청부서 
Q7. 요청하시는 분 성함과 부서가 어디인가요? 

사용자에게 이렇게 8가지의 입력값들을 프론트로 받으면, 연결된 슈퍼베이스 디비 테이블에 칼럼으로 쌓이게 해줘 



이렇게 데이터베이스에 (supabase)에 잘 쌓였다면 
해당 데이터베이스에 있는 칼럼들을 이용해서 피피티 템플릿에 맞게 내용을 채워줄거야
내가 지금 첨부한 피피티 템플릿이랑 supabase 칼럼과의 관계에 대해서 설명해줄게 



(1) PPT 1페이지 
- 해당 페이지는 개발하고자 하는 과제의 제목을 적어주는 '표지' 페이지야 
- supabase 칼럼에서 '과제명'에 해당하는 칼럼을 가져와서 글자크기 40p, 글씨체는 '나눔스퀘어라운드 Regular'로 표지를 그려줘 


(2) PPT 2페이지 
- 해당 페이지는 개발하고자 하는 과제의 개요를 잡아주는 '개요' 페이지야 
- 내가 첨부한 피피티 샘플페이지처 
1) 가장 상단에는 과제명을 적어줘 (supabase의 '과제명'칼럼)
2) 과제명 하단에는 과제설명을 적어줘 (supabase의 '과제설명칼럼') 
3) 과제설명 아래에는 화면 ID와 화면명을 적어줘 (supabase의 '화면명') 
4) 과제명 오른쪽에는 요청부서를 적어줘 
5) 과제설명 오른쪽에는 요청자 이름을 적어줘 
6) 화면ID와 화면명 오른쪽에는 'Description'이라는 칸을 생성해서 
- 첫번째 칸에는 '문제상황'을 적어주고, 이건 supabase의 '요청배경'에 해당하는 부분이야 
- 두번째 칸에는 '변경상황'을 적어주고, 이건 supabase의 '변경사항'에 해당하는 부분이야 
7) Description 좌측에는, 전체 개발사항에 대한 로직 프로세스를 그려주면 되는데
- 템플릿에 적힌 것처럼, 변경 전 로직까지 작성할 필요는 없구, 변경 후 로직만 그려주면 돼 

(3) PPT 3페이지 
- 해당 페이지는 지금까지 설명한 개발요건 사항들을 상세하게 풀어서 설명해주는 페이지인데 
- 해당 페이지에는, 샘플 템플릿처럼 화면ID, 화면명, Description만 적어주면 돼
- 이거는 일단 가능하면 작성해주고, 너가 알아서 이미지나 그런 것들을 첨부해서 적어줘 

내가 지금 첨부하는 피피티랑 완전 똑같은 형태로, Supabase의 개발요청서 자동화 - 입력테이블 값들을 활용해서 
피피티를 작성해주는 로직 만들어줘... 프론트에서 입력받은 값들이 지금 supabase에 쌓여있으니, 백엔드를 만들어주면 돼 

내가 첨부한 피피티랑 색감이나 느낌, 디자인 등등 완전 똑같이 만들어서 줘 
'요청내역' 페이지에 '다운로드' 버튼을 추가해서 해당 개발요청서에 대한 '다운로드' 버튼을 눌렀을 때 
개발요청서를 다운받을 수 있게 해줘
